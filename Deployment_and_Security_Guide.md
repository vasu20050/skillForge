# 🚀 SkillForge: Deployment & Security Blueprint 🛡️

This guide outlines the steps to take your platform from **Localhost → Production** securely and professionally.

---

## 1. Deployment Strategy (Modern Startup Stack)

### Preferred Stack:
- **Frontend**: [Vercel](https://vercel.com) (Best for auto-deploying React/Vite apps).
- **Backend API**: [Railway](https://railway.com) or [Render](https://render.com) (Best for Node.js servers).
- **Database**: [MongoDB Atlas](https://www.mongodb.com/atlas) (Managed Cloud DB already configured).

### Step-by-Step Flow:
1. **Prepare Frontend**:
   - Push your code to **GitHub**.
   - Connect the repository to Vercel.
   - Set the `REACT_APP_API_URL` environment variable in Vercel to your Backend URL.

2. **Prepare Backend**:
   - Create a new Project on Railway/Render.
   - Connect the same GitHub repository.
   - **CRITICAL**: Change the "Root Directory" to `/server`.
   - Set the `PORT`, `MONGO_URI`, and `JWT_SECRET` in the dashboard settings manually. **NEVER push your `.env` file to GitHub.**

3. **Verify Connection**:
   - Update your Backend CORS settings to only allow your Frontend URL (see below).

## 2. Is it Secure to Deploy? (Status: 🟡 Needs Cleanup)

While the core authentication is solid, here is a Checklist to ensure it's **Production-Ready**:

### ✅ What's Secure Now:
- **Password Hashing**: We are using `bcrypt` which is an industry standard.
- **Session Auth**: JWT (JSON Web Tokens) are used for secure state-less authentication.
- **Domain Restriction**: We effectively block non-college emails, preventing random bot registrations.

### ⚠️ Critical Security To-Dos (Before Deploying):

#### 1. Restrict CORS (Server-Side)
Currently, **anyone** can call your API. Update `server/index.js` to restrict access:
```javascript
// Production CORS logic
const corsOptions = {
  origin: process.env.CLIENT_URL || "http://localhost:3000",
  optionsSuccessStatus: 200 
};
app.use(cors(corsOptions));
```

#### 2. Rotate Secrets
You should change your `JWT_SECRET` from `super_secret_skillforge_key_2026` to a random long string generated by a safe generator.

#### 3. Database Whitelisting (Crucial!)
In **MongoDB Atlas Dashboard**, you must:
- Either allow "Access from Anywhere" (`0.0.0.0/0`) if your cloud provider uses dynamic IPs.
- OR (Better) whitelist the specific IP addresses provided by Vercel/Railway.

#### 4. Add Rate Limiting (Prevention against Spam)
Implement `express-rate-limit` to prevent brute force attacks on your `/login` and `/register` endpoints.

#### 5. Helmet Middleware
Add `helmet` to your server to protect against common web vulnerabilities (XSS, Clickjacking, etc.).
```bash
npm install helmet --prefix server
```
Then add to `server/index.js`: `app.use(helmet());`

## 3. High-Impact Performance Tips:
- **Enable Gzip Compression**: Use `compression` middleware on the server to make the API faster.
- **Use HTTP/2**: Most hosting providers (Vercel/Railway) do this by default.
- **Static Asset Caching**: Ensure your logo and images have correct cache headers.

### **Verdict**: 
**Yes, it is secure enough for a MVP/Pilot Program**, but you **MUST** follow the CORS and Secret Rotation steps above to prevent unauthorized access once your URL is public.
